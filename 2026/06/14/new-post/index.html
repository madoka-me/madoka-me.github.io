时隔多年搬迁到Hexo，回归，然后测试

2026-06-14 まどか（Madoka）

用 Antigravity IDE 这个Agent，使用 Claude Opus 4.6 语言模型，自动迭代，把原 Typecho 的 Fantasy 主题重构迁移到 Hexo 了，其中最开始也用过 Gemini 3.1 Pro，然而它并不适合写重构项目，最开始总是自我发挥，不看原项目代码，导致一堆屎山以及连样式都不一致，而且原本项目是无外源资源依赖，是纯本地化的，它却自作主张引入了七牛云等外链导致其他设备无法显示一些图标，而且说到它写的“最近评论”板块，由于Gemini 3.1 Pro 写的渲染和拉取 Github API 的逻辑有问题，导致无法显示，它也没发现，还认为是 Github API 速率问题，然而 Claude 4.6 Opus 只经过两轮调试就发现问题并解决了。

此外，Gemini 3.1 Pro 的记忆力特别差，原因也许是在于它的有效注意力长度太短（网传），表现为只要上下文达到接近100k时，就极易丢三落四，甚至它都忘了之前修改过什么，以及看过的代码和一些领悟的经验，从而越改越屎山，而且它也容易陷入死循环。

比如，Antigravity IDE 是允许 Agent 自行调用 Chrome 进行调试的，它居然连 Hexo 的启动命令都忘记执行，触发数秒超时后，它居然认为是等待时间不够，还加大工具调用等待时间，且几轮如此反复，最终还是不得不由我亲自按下停止键，并向它反馈实际情况和指出它的问题，它才悟了。但是过几轮后，它居然又忘了，不过反复撞墙多次后，它居然又想起来了，但是额度就这样被它耗了不知道多少。

而且 Gemini 3.1 Pro 似乎很不喜欢调用 内置的 Chrome 工具去调试并检验实际效果，以至于每次我一看还是有问题，而 Claude Opus 4.6 恰巧相反，每次改完项目代码就会去完整调试并检验……
